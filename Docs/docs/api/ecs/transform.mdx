# Transform System

```c
#include <safi/ecs/components.h>
```

SafiEngine represents spatial data with `SafiTransform`, a component that stores local position, rotation, and scale. Every entity that needs to exist in 3D space carries one.

## Local transform

```c
typedef struct SafiTransform {
    vec3   position;   // translation
    versor rotation;   // quaternion (x, y, z, w)
    vec3   scale;
} SafiTransform;

static inline SafiTransform safi_transform_identity(void);
```

`safi_transform_identity()` returns zero position, identity quaternion, and unit scale.

## Example

```c
ecs_entity_t e = ecs_new(world);
ecs_set(world, e, SafiTransform, {
    .position = {0, 2, -5},
    .rotation = {0, 0, 0, 1},
    .scale    = {1, 1, 1},
});
```

:::warning WIP — Global transform & hierarchy
The following are not yet implemented:
- **`SafiGlobalTransform`** — no world-space transform component exists; model matrices are built ad hoc in user code
- **Hierarchy propagation** — parent/child relationships exist via flecs, but transforms are not propagated through the hierarchy
- **Change tracking** — no dirty flags or change-driven recomputation; `ecs_modified` must be called manually
:::
