# Render Overview

SafiEngine's renderer is a thin wrapper around the **SDL_gpu** API introduced in SDL 3. SDL_gpu is Khronos-style, explicit, and portable — the same code runs on Metal (macOS/iOS), Vulkan (Linux/Windows/Android), and D3D12 (Windows) without `#ifdef`s.

## Frame lifecycle

```text
safi_renderer_begin_frame(r)
  ├─ SDL_AcquireGPUCommandBuffer
  ├─ SDL_WaitAndAcquireGPUSwapchainTexture
  ├─ (resize depth texture if needed)
  └─ SDL_BeginGPURenderPass  (color + depth)

<user draw calls via SDL_BindGPU…, SDL_DrawGPU…>

safi_debug_ui_render(r)         // ImGui on top
safi_renderer_end_frame(r)
  ├─ SDL_EndGPURenderPass
  └─ SDL_SubmitGPUCommandBuffer
```

Only one render pass per frame in the first-pass engine. Adding more passes (shadow maps, post-processing) is a future evolution.

## Shaders: HLSL → SPIR-V + MSL

Shaders are authored once in **HLSL** and compiled at build time into **SPIR-V** (for Vulkan) and **MSL** (for Metal) via glslangValidator + spirv-cross. At runtime, [`safi_shader_load`](./shader) inspects `SDL_GetGPUShaderFormats()` and picks the matching artifact — callers only pass a logical shader name like `"unlit"`.

See [`cmake/SafiShaders.cmake`](./shader#build-pipeline) for the build pipeline and [Shaders](./shader) for the HLSL binding layout.

## Coordinate conventions

| Axis  | Meaning                                                         |
| ----- | --------------------------------------------------------------- |
| `+X`  | right                                                           |
| `+Y`  | up                                                              |
| `-Z`  | forward                                                         |
| Wind. | counter-clockwise = front (SDL_GPU_FRONTFACE_COUNTER_CLOCKWISE) |
| Depth | `SDL_GPU_COMPAREOP_LESS`, `clear = 1.0`                         |

These match cglm's `glm_lookat` / `glm_perspective` defaults, so you can hand camera matrices straight to the shader without sign flips.
