# Asset System

:::warning WIP — Not yet implemented
The engine does not have an asset management layer. GPU resources are created and destroyed manually, and components store raw pointers.
:::

## Current state

- `SafiMeshRenderer` holds `SafiMesh*` and `SafiMaterial*` — raw pointers with no reference counting
- `SafiMaterial` owns an `SDL_GPUTexture*` directly
- `safi_gltf_load` performs synchronous parse, decode, and GPU upload in one call
- Loading the same file twice creates duplicate GPU resources

## Planned features

### Asset handles

Opaque handle types (e.g. `SafiHandle(SafiMesh)`) that reference assets by ID rather than pointer. Handles can be copied freely and remain valid as long as the asset server owns the underlying resource.

### Asset server / registry

A central `SafiAssetServer` resource that owns all loaded assets, maps paths to handles, and manages lifetimes. Components store handles; the render system resolves handles to GPU resources at draw time.

### Caching & deduplication

The asset server tracks loaded paths. Requesting an already-loaded asset returns the existing handle instead of duplicating GPU resources.

### Async / staged loading

A background loading path that parses and decodes assets off the main thread, then uploads to the GPU in a staging step. This prevents frame hitches when loading large models or textures.
