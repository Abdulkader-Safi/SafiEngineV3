# Debug UI

SafiEngine's debug overlay uses **[Nuklear](https://github.com/Immediate-Mode-UI/Nuklear)** — a single-header, pure-C immediate-mode UI library. The engine ships its own SDL_gpu backend (font atlas upload, vertex/index buffers, pipeline, scissor + draw) so the whole stack stays in C.

```c
#include <safi/ui/debug_ui.h>
```

## Lifecycle

The debug UI is opt-in via `SafiAppDesc.enable_debug_ui`. When enabled:

1. `safi_app_init` calls `safi_debug_ui_init` — creates the Nuklear context, bakes the default font into an `SDL_GPUTexture`, builds the UI graphics pipeline.
2. Each frame, after `safi_renderer_begin_frame` and **before** `safi_renderer_begin_main_pass`:
   - `safi_debug_ui_begin_frame(r)` — opens the widget window.
   - `safi_debug_ui_draw_panels(r, world)` — draws the built-in Scene and Inspector panels.
   - Optionally build additional custom widgets with `nk_begin / nk_layout_row_dynamic / …`.
   - `safi_debug_ui_prepare(r)` — runs `nk_convert` and uploads vertices/indices through a copy pass. Must happen before the main render pass opens; SDL_gpu forbids nested passes.
3. After `safi_renderer_begin_main_pass` and any engine geometry, call `safi_debug_ui_render(r)` to record the Nuklear draw commands into the active render pass.
4. `safi_app_shutdown` calls `safi_debug_ui_shutdown`.

The engine's input system already forwards SDL events to Nuklear — you don't need to call anything extra for mouse, keyboard, or text input.

## Built-in panels

The engine provides two ready-to-use panels via `safi_debug_ui_draw_panels`. No Nuklear include or widget code needed in your app.

### Scene hierarchy

Positioned on the **left side** of the window. Lists every entity that has a `SafiName` component. Clicking an entry selects it for the Inspector.

### Inspector

Positioned on the **right side** of the window. Displays the selected entity's name and all its stock components with editable fields:

| Component          | Fields shown                                              |
| ------------------ | --------------------------------------------------------- |
| `SafiTransform`    | Position (x/y/z), Rotation (euler degrees), Scale (x/y/z) |
| `SafiCamera`       | FOV (degrees), Near, Far, Target (x/y/z)                  |
| `SafiMeshRenderer` | Mesh and Material assignment status                       |
| `SafiSpin`         | Speed (rad/s), Axis (x/y/z)                               |

Only components present on the selected entity are shown.

### Enabling / disabling

The panels are gated by `SafiAppDesc.enable_debug_ui`:

```c
SafiAppDesc desc = {
    .title           = "My Game",
    .width           = 1280,
    .height          = 720,
    .enable_debug_ui = true,   // set to false to disable all debug UI
};
```

At runtime, `app.debug_ui_enabled` can be toggled to show/hide the panels.

## Functions

### Framework

```c
bool safi_debug_ui_init(SafiRenderer *r);
void safi_debug_ui_shutdown(SafiRenderer *r);
void safi_debug_ui_process_event(const void *sdl_event);  // called by the engine
void safi_debug_ui_begin_frame(SafiRenderer *r);
void safi_debug_ui_prepare(SafiRenderer *r);             // pre-pass, uploads UI vbo/ibo
void safi_debug_ui_render(SafiRenderer *r);              // in-pass, records draws
struct nk_context *safi_debug_ui_context(void);          // the underlying nk context
bool safi_debug_ui_wants_input(void);                    // true when a widget is active
```

### Scene & Inspector

```c
void         safi_debug_ui_draw_panels(SafiRenderer *r, ecs_world_t *world);
ecs_entity_t safi_debug_ui_selected_entity(void);
void         safi_debug_ui_select_entity(ecs_entity_t e);
```

| Function                        | Description                                                                                 |
| ------------------------------- | ------------------------------------------------------------------------------------------- |
| `safi_debug_ui_draw_panels`     | Draws the Scene hierarchy and Inspector windows. Call between `begin_frame` and `prepare`.  |
| `safi_debug_ui_selected_entity` | Returns the entity currently shown in the Inspector.                                        |
| `safi_debug_ui_select_entity`   | Sets the entity shown in the Inspector. Typically called once at startup to pick a default. |

## Example

```c title="main.c"
#include <safi/safi.h>

static void render_system(ecs_iter_t *it) {
    SafiApp *app = (SafiApp *)it->ctx;
    SafiRenderer *r = &app->renderer;

    if (!safi_renderer_begin_frame(r)) return;

    if (app->debug_ui_enabled) {
        safi_debug_ui_begin_frame(r);
        safi_debug_ui_draw_panels(r, it->world);
        safi_debug_ui_prepare(r);
    }

    safi_renderer_begin_main_pass(r);
    // ... draw your scene ...
    if (app->debug_ui_enabled) safi_debug_ui_render(r);
    safi_renderer_end_main_pass(r);
    safi_renderer_end_frame(r);
}
```

## Custom widgets

If you need additional Nuklear widgets beyond the built-in panels, include `<nuklear.h>` with the engine's feature macros and add your own `nk_begin` / `nk_end` blocks between `safi_debug_ui_begin_frame` and `safi_debug_ui_prepare`:

```c
#define NK_INCLUDE_FIXED_TYPES
#define NK_INCLUDE_STANDARD_IO
#define NK_INCLUDE_STANDARD_VARARGS
#define NK_INCLUDE_DEFAULT_ALLOCATOR
#define NK_INCLUDE_VERTEX_BUFFER_OUTPUT
#define NK_INCLUDE_FONT_BAKING
#define NK_INCLUDE_DEFAULT_FONT
#include <nuklear.h>
```

:::warning Feature macros must match
The engine's `debug_ui.c` defines `NK_IMPLEMENTATION` together with the seven `NK_INCLUDE_*` macros above. Your translation unit must define **exactly the same** set (without `NK_IMPLEMENTATION`) so the `nk_context` struct layout matches across TUs.
:::

## Why Nuklear (not Dear ImGui)?

An earlier iteration used cimgui to drive Dear ImGui. That worked but required a C++ translation unit (`cmake/cimgui_bridge.cpp`) just to `extern "C"` the ImGui SDL3+SDL_gpu backends, plus a separate `safi_cimgui` static library compiling ~8 imgui `.cpp` files. For a **pure C** engine it added too much surface area.

Nuklear is:

- **a single header**, no generator, no submodule, no C++
- **pure C99**, so `safi_engine` is now 100% C with **zero** `.cpp` files compiled under its target
- license-friendly (MIT / public domain dual-licensed)
- dependency-free (uses its own vendored `stb_truetype` and `stb_rect_pack` for font baking)

Trade-off: Nuklear has no docking / multi-viewport and a smaller widget catalogue. For a debug overlay this is fine.
